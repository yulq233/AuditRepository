# 抽样检查模块设计文档

## 1. 文档概述

本文档旨在设计一款面向审计领域的AI辅助抽样检查模块。该模块与AI抽凭助手深度集成，实现从总体定义、样本抽取、自动化测试到结果推断的全流程智能化，帮助审计人员高效、准确地完成抽样测试工作。模块遵循审计准则中的统计抽样与非统计抽样方法，并通过AI技术提升样本测试的自动化程度与风险识别能力。

---

## 2. 功能模块清单

| 功能模块 | 具体功能 | 说明 |
|---------|---------|------|
| **抽样计划配置** | 总体定义 | 支持从业务系统/财务系统导入总体数据（如应收账款明细、凭证列表）；支持按科目、期间、金额范围、客户等条件筛选总体。 |
| | 抽样方法选择 | 统计抽样（货币单位抽样MUS、系统抽样、随机抽样）或非统计抽样（定向抽样、任意抽样）。 |
| | 参数设置 | 设置置信水平（如95%）、可容忍错报率、预期错报率、总体金额/笔数等。自动计算样本量。 |
| | 分层设置 | 支持按金额、重要性、风险等级对总体分层，并为每层单独设置抽样参数。 |
| **样本生成与管理** | 自动抽取样本 | 根据抽样方法自动生成样本清单，记录每个样本的抽取逻辑和权重。 |
| | 手动调整样本 | 允许审计师手动替换或追加样本，系统记录调整原因。 |
| | 样本执行状态 | 跟踪每个样本的执行状态（待测试、测试中、已完成）。 |
| **样本测试（AI驱动）** | AI抽凭助手集成 | 调用AI抽凭助手对每个样本进行自动化证据链核验（合同、发票、物流单、银行回单等），输出样本是否真实合理，并提取关键信息。 |
| | 人工复核与修正 | 对AI判定结果提供人工复核入口，允许审计师修改样本结果（如将AI判定为“异常”但实为合理的样本标记为“正常”）。 |
| | 样本错报记录 | 记录每个样本发现的错报类型、金额、原因。支持上传错报证据截图。 |
| **结果统计与推断** | 样本错报汇总 | 自动汇总样本中的错报金额、错报笔数、错报率。 |
| | 总体推断 | 根据统计抽样公式推断总体错报上限、点估计值，生成统计结论。 |
| | 扩展测试建议 | 若推断结果超过可容忍错报，系统建议执行进一步程序（如扩大样本量、执行替代程序）。 |
| **报告与底稿** | 抽样报告生成 | 一键生成抽样计划、样本执行记录、错报明细、统计推断结果的Word/PDF报告。 |
| | 底稿导出 | 将抽样过程、样本测试证据链、错报分析自动归入审计底稿。 |

## 3. 抽样检查全流程

以下流程详细描述了从定义总体到生成结论的每一步操作，并标注了AI调用时机及后处理逻辑。

### 3.1 步骤1：定义抽样总体
- **操作**：审计师在系统中选择要测试的会计科目（如应收账款）或交易类型（如销售发票），设定期间（如2024年1-12月）。系统从企业ERP或财务系统中抽取符合条件的所有凭证或交易记录，形成总体清单。
- **AI调用时机**：此时无需调用AI，但系统可利用AI对总体进行预处理，例如：
  - 对每笔交易进行初步风险评分（如基于历史数据、金额大小、客户信用等），为后续分层或抽样提供依据。
- **输出**：总体列表（含金额、交易日期、客户名称等字段），总体总金额、总笔数。

### 3.2 步骤2：设定抽样参数
- **操作**：审计师根据审计目标选择抽样方法（统计或非统计），并设置参数。
  - **统计抽样（以货币单位抽样为例）**：
    - 设定置信水平（默认95%）、可容忍错报率（如5%）、预期错报率（如0.5%）。
    - 系统自动计算样本量（可参考公式：n = 总体金额 / 抽样间隔，抽样间隔 = 可容忍错报 / 保证系数）。
  - **非统计抽样**：
    - 审计师基于职业判断，设定需要抽取的样本数量或金额比例。
- **AI调用时机**：AI可根据历史类似项目的抽样结果或行业基准，推荐合理的参数设置（例如“根据过去三年该客户的错报率，建议预期错报率设为0.8%”）。
- **输出**：抽样计划参数表。

### 3.3 步骤3：分层与样本抽取
- **操作**：
  1. **分层**：若总体金额分布不均（如大量小额交易+少量大额交易），审计师可启用分层功能，按金额或风险等级划分若干层。系统自动为每层分配样本量（通常金额越高、风险越大的层，抽取比例越高）。
  2. **抽取样本**：
     - **统计抽样**：系统根据抽样方法自动抽取样本。例如货币单位抽样时，系统生成随机起点和抽样间隔，在总体金额累计数中选取样本。
     - **非统计抽样**：系统根据审计师指定的规则（如金额最大的前N笔、随机抽取等）生成样本。
- **AI调用时机**：AI可以分析总体中各笔交易的风险特征（如是否有异常交易模式、关联方交易等），辅助审计师优化分层策略（例如将AI识别的高风险交易自动划入“重点层”）。
- **输出**：样本清单（含每个样本的标识、金额、客户名称、抽取依据）。

### 3.4 步骤4：样本执行与AI测试
- **操作**：
  1. **启动AI测试**：审计师在系统中选择“执行样本测试”，系统将样本清单发送至AI抽凭助手。
  2. **AI自动化处理**：
     - 对每个样本，AI抽凭助手自动获取相关单据（合同、发票、物流单、银行回单等），执行多模态识别与证据链校验。
     - 输出结果：每个样本的判定结论（“真实合理” / “存在异常” / “无法判断”）、异常描述（如金额不符、日期倒挂、第三方回款等）、提取的关键字段。
  3. **结果回传**：AI将每个样本的测试结果写入抽样模块。
- **AI调用时机**：此处大量调用AI服务，执行异步任务。系统需处理AI返回的结构化结果。
- **后处理**：
  - 对AI判定为“存在异常”的样本，系统自动标记为“疑似错报”。
  - 对AI判定为“无法判断”的样本（如单据缺失、模糊不清），系统将其标记为“需人工复核”。
- **输出**：每个样本的测试状态和结果。

### 3.5 步骤5：人工复核与修正
- **操作**：
  1. 审计师对AI判定为“异常”或“无法判断”的样本进行人工复核。
  2. 复核时，审计师可查看AI提取的证据链、原始单据，并决定是否修正AI的判定（例如将“异常”修正为“合理”，并备注理由）。
  3. 若发现错报，审计师记录错报金额、错报类型（如虚构交易、金额错误、未入账等）。
- **AI调用时机**：无需再调用AI，但系统应记录人工修正的行为，用于后续AI模型优化（如将修正后的样本作为训练数据）。
- **输出**：每个样本的最终错报认定（错报金额/无错报）。

### 3.6 步骤6：样本结果汇总
- **操作**：系统自动统计以下指标：
  - 样本总金额、样本总笔数。
  - 错报样本数、错报总金额、错报比率。
  - 按错报类型分类统计（如金额错误、缺少支持性单据等）。
- **AI调用时机**：无。
- **输出**：样本错报汇总表。

### 3.7 步骤7：总体推断（统计抽样）
- **操作**：
  - 若抽样方法为统计抽样，系统自动执行总体推断。
    - **货币单位抽样**：根据错报样本的错报比例、错报金额，计算总体错报上限（如使用泊松分布或正态分布方法），并与可容忍错报比较。
    - **系统抽样**：使用样本错报率来推断总体错报率，并计算置信区间。
  - 系统输出推断结论：“总体错报未超过可容忍错报”或“总体错报可能超过可容忍错报”。
- **AI调用时机**：无。
- **后处理**：若推断结论为“可能超过可容忍错报”，系统自动建议审计师采取进一步措施（如扩大样本量、执行替代程序、提请管理层调整账目）。

### 3.8 步骤8：形成结论与报告
- **操作**：
  1. 审计师根据推断结果、非统计抽样时的职业判断，最终确定对总体是否可接受的结论。
  2. 一键生成抽样检查报告，包含：
     - 抽样计划描述（总体、方法、参数）
     - 样本清单及测试结果（含AI证据截图）
     - 错报明细及汇总
     - 统计推断过程与结论
     - 审计结论（如“未发现重大错报”或“存在重大错报风险，建议调整”）
  3. 导出审计底稿（Word/Excel），自动归档至审计项目文档。
- **AI调用时机**：无，但系统可利用AI生成报告摘要或自然语言描述，提升报告可读性。

## 4. 抽样策略定义详解

抽样策略由审计师在步骤2中定义，系统需提供以下配置界面：

### 4.1 抽样方法选择
- **统计抽样**：货币单位抽样（MUS）、系统抽样、随机抽样。
- **非统计抽样**：金额分层抽样、重要性抽样、随机抽取、特定项目抽样（如大额、异常交易）。

### 4.2 关键参数
- **货币单位抽样**：
  - 置信水平（例如95%、90%）
  - 可容忍错报率（相对于总体金额的百分比，或绝对值）
  - 预期错报率（若预期错报为0，可简化抽样）
- **系统抽样**：
  - 抽样间隔 = 总体笔数 / 样本量
  - 随机起点
- **非统计抽样**：
  - 样本量（笔数或金额比例）
  - 抽取规则（如金额前20笔、随机10笔等）

### 4.3 分层策略
- 分层依据：按金额区间、风险评分（AI可提供）、交易类型等。
- 每层单独设置抽样比例或样本量。

### 4.4 AI辅助优化
- 系统可基于历史项目数据训练模型，为当前项目推荐最佳抽样方法、样本量或分层阈值。审计师可选择采纳或手动修改。

## 5. AI结果后处理逻辑

AI抽凭助手返回每个样本的测试结果，抽样模块需对结果进行以下处理：

1. **解析与存储**：
   - 将AI返回的判定结论（如“真实合理”、“存在异常”、“无法判断”）映射到系统内部状态。
   - 提取AI给出的异常描述、关键字段、证据截图位置，存入数据库。

2. **错报识别与统计**：
   - 若AI判定为“存在异常”，系统自动将其标记为“疑似错报”，并计入样本错报统计（除非人工复核后修正）。
   - 若AI提供了错报金额（如发票金额与合同金额差异），系统自动累加该样本的错报金额；若AI未提供错报金额，审计师需在人工复核时补录。

3. **人工复核工作流**：
   - 对于AI无法判断的样本，系统自动创建待办任务，分配给指定审计师。
   - 审计师复核后，可将样本状态修改为“正常”或“错报”，并填写错报金额。系统记录修改前后差异，用于模型迭代。

4. **影响后续抽样**：
   - 在扩大样本量的情况下，AI结果可帮助审计师决定扩大范围。例如，若AI发现某个子总体（如某客户）错报集中，系统可建议对该客户相关交易追加测试。
   - 系统可将AI检测出的异常交易特征（如关联方回款）作为后续非统计抽样的筛选条件。

5. **模型反馈闭环**：
   - 抽样模块记录所有人工复核的结果，定期导出为训练数据集，用于AI模型的持续优化（例如，提升对复杂异常模式的识别能力）。

---

## 6. 总结

AI辅助抽样检查模块将传统审计抽样流程与AI自动化测试深度结合，实现了：
- **效率提升**：AI自动完成样本测试，释放审计师时间。
- **准确性增强**：通过规则引擎和大模型减少人为遗漏。
- **可追溯性**：所有抽样决策、AI判定、人工复核均留痕，满足审计准则要求。

该模块可作为独立产品功能，也可与AI抽凭助手无缝集成，构成完整的审计智能化解决方案。开发时需重点关注抽样参数的计算准确性、AI接口的稳定性、以及人工复核流程的易用性。
