## 安全编码约束（强制）

生成任何代码时，必须遵循以下安全实践：

### 输入处理
- 所有用户输入必须验证和清洗（类型、长度、格式）
- SQL 查询必须使用参数化查询，禁止字符串拼接
- HTML 输出必须转义，防止 XSS
- 文件路径操作必须验证防止路径遍历

### 认证与授权
- 密码必须使用强哈希算法（bcrypt/Argon2），禁止 MD5/SHA1
- API 端点必须有明确的认证检查
- 默认拒绝所有权限，显式授权例外

### 敏感数据
- 禁止硬编码 API 密钥、密码、证书
- secrets 使用环境变量或密钥管理服务
- 日志中不得出现 PII、密码、令牌

### 错误处理
- 错误消息不暴露内部实现细节
- 所有异常必须被捕获并记录
- 对外错误消息使用通用提示

### 依赖安全
- 添加新依赖时说明其来源和版本
- 优先使用维护中的依赖，避免已知漏洞版本
- 重要外部资源包含完整性校验

### 对齐标准
- 代码建议必须符合 OWASP Top 10 原则
- 对于 Python 项目：使用 bandit 规则集作为参照
- 参考 CWE/SANS Top 25 常见安全弱点
